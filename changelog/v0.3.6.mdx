Task lifecycle hooks, custom template registries, parent-child subtasks, and a critical hook fix.

## Enhancements

- **Custom template registry** (`--registry`): Pull spec templates from your own GitHub, GitLab, or Bitbucket repo during `trellis init`. Supports marketplace mode (with `index.json`) and direct download mode. See [Remote Spec Templates](/guide/ch02-quick-start#25-remote-spec-templates).
- **Task lifecycle hooks**: Configure `after_create`, `after_start`, `after_finish`, and `after_archive` hooks in `.trellis/config.yaml`. Each hook receives `TASK_JSON_PATH` as an environment variable. Ships with a Linear sync hook example. See [Task Lifecycle Hooks](/guide/ch06-task-management#66-task-lifecycle-hooks).
- **Parent-child subtasks**: `add-subtask` and `remove-subtask` commands, `--parent` flag on `task.py create`, new `children`/`parent`/`meta` fields in `task.json`.
- **record-session improvements**: Archive judgment now based on actual work status, not the `status` field in `task.json`.
- **brainstorm subtask decomposition**: `/trellis:brainstorm` Step 8 now includes subtask creation for complex tasks.

## Bug Fixes

- **PreToolUse context injection broken by Claude Code v2.1.63**: CC renamed the internal tool from `Task` to `Agent` ([anthropics/claude-code#29677](https://github.com/anthropics/claude-code/issues/29677)), silently breaking the PreToolUse hook — `implement`, `check`, `debug`, and `research` agents were not receiving injected code-spec context. Fixed by accepting both `Task` and `Agent` as valid tool names.

<Warning>
  The PreToolUse bug affected **all** Trellis users on Claude Code v2.1.63+. Run `trellis update` to
  apply the fix.
</Warning>

<Note>Run `trellis update` to sync updated hook scripts and command templates.</Note>
