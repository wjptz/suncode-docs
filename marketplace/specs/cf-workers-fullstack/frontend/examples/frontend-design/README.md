# Frontend Design Skill Examples

This directory contains example implementations demonstrating how to apply the frontend-design skill principles in this project.

## Overview

These examples show how to create distinctive, production-grade frontend interfaces using:

- React + TypeScript
- TailwindCSS
- shadcn/ui components
- Cloudflare Workers compatible patterns

## Examples

### 1. Minimalist Landing Page

**File**: `minimal-hero.tsx.template`

- Clean, refined aesthetic with generous white space
- Precision typography and subtle animations
- Demonstrates restraint and attention to detail

### 2. Maximalist Dashboard

**File**: `maximalist-dashboard.tsx.template`

- Rich visual layers with gradients, textures, and patterns
- Complex animations and staggered reveals
- Shows how to create atmospheric depth

### 3. Custom Font Configuration

**File**: `tailwind-fonts.config.template`

- Example Tailwind config with distinctive font pairings
- Shows how to avoid generic font choices
- Demonstrates font utility setup

### 4. Animation Library

**File**: `animations.css.template`

- Reusable CSS animations for high-impact moments
- Staggered reveal patterns
- Scroll-triggered and hover effects

## Usage

1. **Choose an aesthetic direction** from the Design Philosophy section
2. **Review relevant examples** that match your chosen direction
3. **Adapt the patterns** to your specific use case
4. **Maintain type safety** and project conventions
5. **Test accessibility** and performance

## Key Principles

- **No Generic Aesthetics**: Avoid Inter, Roboto, purple gradients, cookie-cutter layouts
- **Context-Specific**: Design for the specific problem, not generic templates
- **Bold Commitment**: Fully execute the chosen aesthetic direction
- **Production-Ready**: All code must be functional, accessible, and performant

## File Naming Convention

All code examples use `.template` suffix to avoid IDE errors and make it clear these are reference implementations, not production code.
