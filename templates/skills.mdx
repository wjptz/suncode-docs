## What are skills?

Skills are bundles of knowledge that Claude loads when relevant. Unlike commands (which you invoke explicitly), skills activate automatically when the AI recognizes they apply.

A skill might cover:

- A specific technology (PostgreSQL, Redis)
- A workflow (code review, debugging)
- A domain (API design, security)

## Skill structure

Skills live in `.claude/skills/`:

```
.claude/skills/
└── my-skill/
    └── SKILL.md
```

The `SKILL.md` file contains frontmatter and content:

```markdown
---
name: my-skill
description: |
  Brief description of what this skill covers.
  Claude uses this to decide when to load the skill.
---

# Skill Content

The actual knowledge goes here. Write it like you're explaining to a competent developer.
```

## Template: Database skill

`.claude/skills/postgres/SKILL.md`:

````markdown
---
name: postgres
description: |
  PostgreSQL database patterns and best practices.
  Use when working with Postgres, writing SQL, or database migrations.
---

# PostgreSQL Guidelines

## Query patterns

Use parameterized queries. Never interpolate user input:

```sql
-- Wrong
SELECT * FROM users WHERE id = '${userId}'

-- Right
SELECT * FROM users WHERE id = $1
```
````

## Indexing

Add indexes for:

- Foreign keys
- Columns in WHERE clauses
- Columns in ORDER BY

```sql
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_orders_user_id ON orders(user_id);
```

## Migrations

One change per migration. Make migrations reversible:

```sql
-- up
ALTER TABLE users ADD COLUMN phone VARCHAR(20);

-- down
ALTER TABLE users DROP COLUMN phone;
```

## JSON columns

Use `jsonb`, not `json`. Index JSON paths you query often:

```sql
CREATE INDEX idx_settings_theme ON user_settings((settings->>'theme'));
```

## Transactions

Wrap related operations in transactions:

```typescript
await db.transaction(async (tx) => {
  await tx.users.update(userId, { balance: balance - amount });
  await tx.transfers.create({ from: userId, amount });
});
```

````

## Template: Security skill

`.claude/skills/security/SKILL.md`:

```markdown
---
name: security
description: |
  Web application security practices.
  Use when handling auth, user input, or sensitive data.
---

# Security Guidelines

## Input validation

Validate ALL user input. Trust nothing from the client.

```typescript
// Validate with Zod
const UserInput = z.object({
  email: z.string().email(),
  age: z.number().int().min(0).max(150),
});

const result = UserInput.safeParse(req.body);
if (!result.success) {
  throw new ValidationError(result.error);
}
````

## SQL injection

Use parameterized queries. If you must build dynamic SQL, whitelist allowed values:

```typescript
const ALLOWED_SORT = ['created_at', 'name', 'email'];
if (!ALLOWED_SORT.includes(sortField)) {
  throw new Error('Invalid sort field');
}
```

## XSS prevention

Escape output. Use framework defaults. Never use `dangerouslySetInnerHTML` with user content.

## Authentication

- Hash passwords with bcrypt (cost factor 12+)
- Use secure, httpOnly cookies for sessions
- Implement rate limiting on login endpoints
- Token expiration: access tokens short (15min), refresh tokens longer (7 days)

## Secrets

- Never commit secrets to git
- Use environment variables
- Rotate secrets if exposed
- Different secrets per environment

## HTTPS

- Always use HTTPS in production
- Set HSTS headers
- Redirect HTTP to HTTPS

````

## Template: Testing skill

`.claude/skills/testing/SKILL.md`:

```markdown
---
name: testing
description: |
  Testing patterns and practices.
  Use when writing tests, setting up test infrastructure, or debugging test failures.
---

# Testing Guidelines

## Test structure

One behavior per test. Name tests clearly:

```typescript
// Bad
test("user", () => { ... });

// Good
test("createUser returns user with generated ID", () => { ... });
test("createUser throws when email already exists", () => { ... });
````

## Arrange-Act-Assert

```typescript
test('transferMoney updates both balances', async () => {
  // Arrange
  const sender = await createUser({ balance: 100 });
  const receiver = await createUser({ balance: 50 });

  // Act
  await transferMoney(sender.id, receiver.id, 30);

  // Assert
  expect(await getBalance(sender.id)).toBe(70);
  expect(await getBalance(receiver.id)).toBe(80);
});
```

## Mocking

Mock external dependencies, not internal code:

```typescript
// Mock the HTTP client, not the service that uses it
jest.mock('./lib/http');

// Don't mock internal functions just to make tests easier
```

## Test data

Use factories for test data:

```typescript
const createTestUser = (overrides = {}) => ({
  id: randomId(),
  email: `test-${randomId()}@example.com`,
  name: 'Test User',
  ...overrides,
});
```

## Integration tests

Test real database operations. Use transactions for cleanup:

```typescript
beforeEach(async () => {
  await db.beginTransaction();
});

afterEach(async () => {
  await db.rollbackTransaction();
});
```

```

## Installing skills

1. Create the folder: `mkdir -p .claude/skills/skill-name`
2. Create `SKILL.md` with frontmatter and content
3. Skills auto-load when Claude determines they're relevant

For community skills, copy them manually into your `.claude/skills/` directory.

---

## Creating your own

Start from a template above, then:

1. Focus on your project's specific patterns
2. Include actual code from your codebase
3. Be specific - vague skills don't help
4. Test by asking Claude about topics the skill covers
```
