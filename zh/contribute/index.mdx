## 用 AI 辅助贡献

最简单的贡献方式是使用 Claude Code。我们内置了一个 skill 来引导你完成贡献流程。

### 第一步：克隆文档仓库

```bash
git clone https://github.com/mindfold-ai/docs.git
cd docs
```

### 第二步：告诉 Claude 你想贡献什么

在项目中打开 Claude Code，描述你想添加的内容：

- "我想添加一个 Next.js 项目的规范模板"
- "我想改进并行会话的文档"

### 第三步：Claude 自动使用 contribute skill

Claude 会读取 `contribute` skill 并引导你：

- 文件应该放在哪里
- 哪些关联文件需要更新（docs.json、索引页面）
- 双语要求（EN + ZH）
- 如何本地测试

### 第四步：审查并提交 PR

审查修改，用 `pnpm dev` 测试，然后提交 PR。

<Tip>contribute skill 了解项目结构和规范。它处理细节，你只需专注于内容。</Tip>

## 贡献方式

Trellis 是开源项目。欢迎贡献。

### 报告 Bug

发现问题？在 GitHub 开 issue：

https://github.com/mindfold-ai/Trellis/issues

包含：

- 你做了什么
- 你期望什么结果
- 实际发生了什么
- Trellis 版本（`trellis --version`）
- 操作系统和 Claude Code 版本

### 建议功能

有想法？开一个 discussion：

https://github.com/mindfold-ai/Trellis/discussions

描述你想解决的问题，而不只是解决方案。也许有更好的方式，或者功能已经存在。

### 分享模板

做了有用的规范、命令或技能？分享出来。

1. Fork 仓库
2. 把你的模板加到对应的 `templates/` 文件夹
3. 包含一个 README 解释它做什么
4. 开 PR

我们会审查模板质量。好模板：

- 具体可执行
- 文档完善
- 在真实项目上测试过

### 改进文档

文档在 `docs/` 文件夹。修错别字、说清楚含糊的地方、加例子。

小修复：直接在 GitHub 上编辑。
大改动：fork 然后 PR。

### 代码贡献

改代码：

1. 查现有 issue 避免重复工作
2. 开 issue 描述你想改什么
3. 等反馈再开始（省得方向要调整时白做）
4. Fork、建分支、写代码、测试
5. 开 PR

## 代码风格

Trellis 用：

- TypeScript 写 CLI
- Python 写 hooks
- Markdown 写文档和规范

遵循现有模式。提交前跑 linter。

## 测试

提交前跑测试：

```bash
pnpm test
```

加功能的话也要加测试。

## 文档

改了行为就更新文档。过时的文档比没文档更糟。

## Commit 信息

用 conventional commits：

```
feat: add support for custom hook scripts
fix: resolve path issues on Windows
docs: clarify spec loading order
```

## PR 流程

1. 创建 PR，写清楚描述
2. 等审查
3. 处理反馈
4. 批准后合并

PR 通常几天内会审查。如果等太久，在 PR 里 ping 一下。

## 行为准则

保持尊重。这是小项目，由人维护。假设善意。建设性地表达不同意见。

## 许可证

贡献采用和 Trellis 相同的许可证（MIT）。贡献即表示同意。
