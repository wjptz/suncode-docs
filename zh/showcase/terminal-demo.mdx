## 刚才发生了什么？

这个演示回放了一次真实的 Trellis 会话: 我们为 Trellis CLI 添加了 **Gemini CLI 平台支持**，涉及类型定义、模板、配置器、CLI 参数、Python 适配器和文档。

### 工作流程

<Steps>
  <Step title="启动会话">
    `/trellis:start` 加载项目上下文: 工作流规则、活跃任务、Git 状态和最近的工作日志。AI 立即进入状态。
  </Step>
  <Step title="描述需求">
    用自然语言描述你想要的功能。Trellis 创建一个带结构化 PRD 的跟踪任务。
  </Step>
  <Step title="调研与配置">
    trellis-research sub-agent 读取 36 个文件，找到相关的规范和代码模式。上下文文件配置到 jsonl 中，Agent 通过 hooks 自动接收正确的规范。
  </Step>
  <Step title="实现">
    trellis-implement sub-agent 跨 5 层编写代码（类型 → 模板 → 配置器 → CLI → Python）。99 次工具调用，337 个测试一次通过。
  </Step>
  <Step title="质量检查">
    trellis-check sub-agent 对照代码规范审查每个改动文件。发现 1 个缺失的 `EXCLUDE_PATTERNS` 条目并自动修复。
  </Step>
  <Step title="更新规范">
    `/trellis:update-spec` 将本次会话中学到的新模式沉淀到规范库: 让未来的会话更准确。
  </Step>
  <Step title="记录与交付">
    `/trellis:finish-work` 将会话归档到工作日志。5 个原子提交已在功能分支上，准备好等待 Review。
  </Step>
</Steps>

### 关键数据

| 指标 | 数值 |
|------|------|
| **总耗时** | ~20 分钟 |
| **工具调用** | 169 次（explore + research + implement + check） |
| **改动文件** | 14 个 TOML 模板 + 5 个源文件 |
| **测试** | 337/337 通过 |
| **提交** | 5 个原子提交 |
| **人工输入** | 3 条消息（需求描述 + update-spec + finish-work） |
