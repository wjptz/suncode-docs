智能 update 保护、session-start 任务感知、改进 start 流程。

## 新增功能

- **Update：用户删除保护**：如果你主动删除了 Trellis 安装的某个文件，`trellis update` 现在会通过存储的 hash 检测到，不再重新添加。update 输出中新增「Deleted by you (preserved)」分区
- **Update：`update.skip` 配置**：在 `.trellis/config.yaml` 中添加 `update.skip` 列表，可永久排除指定文件或目录不被 `trellis update` 更新。适用于 monorepo 项目中不需要某些平台特定命令的场景
- **Session-start：任务状态注入**：session-start hook 现在注入 `<task-status>` 标签，包含结构化状态（`NO ACTIVE TASK` / `NOT READY` / `READY` / `COMPLETED`），使 AI 能自动检测并恢复进行中的任务
- **Session-start：动态 spec 发现**：session-start hook 现在动态遍历 `spec/` 子目录，不再硬编码 `frontend/backend/guides`，支持 monorepo 包结构（如 `spec/cli/backend/`）
- **Start 流程：强制 brainstorm**：复杂任务现在会在所有 9 个支持平台自动触发 brainstorm 流程，防止在需求不明确时直接进入实现

## 迁移

无需手动操作。运行 `trellis update` 即可同步更新后的 hook 脚本和命令模板。
