## 附录 E：`worktree.yaml`（已移除）

本附录原先描述 `.trellis/worktree.yaml`，即 Multi-Agent Pipeline 的配置文件。该 pipeline 和这个文件都在 **0.5.0-beta.0** 随 `/parallel` 命令、`dispatch` agent、Ralph Loop 一起移除。

如果你因为维护旧版（0.5 前）项目来查老内容，之前的附录 E 在 Release track 文档里保留（仍对应 0.4.0）。

如果你在用旧版的 `verify:` 字段来强制对 AI 输出跑 lint / typecheck / test，替代方案：

- 把同样的命令写进 package spec（如 `.trellis/spec/backend/quality.md`）。
- `trellis-check` sub-agent 读这份 spec，在每次 diff 上跑同样的检查，自带自修复循环。

不涉及任何 Trellis 配置文件。
