## 附录 D：JSONL 配置格式参考

### 文件条目

```jsonl
{"file": ".trellis/spec/backend/index.md", "reason": "后端开发指南"}
```

### 目录条目

```jsonl
{"file": "src/services/", "type": "directory", "reason": "现有 service 模式"}
```

目录条目下 hook 会读目录里所有 `.md`，最多 20 个。

### 完整示例（fullstack `implement.jsonl`）

```jsonl
{"file": ".trellis/workflow.md", "reason": "项目工作流和约定"}
{"file": ".trellis/spec/shared/index.md", "reason": "共享编码标准"}
{"file": ".trellis/spec/backend/index.md", "reason": "后端开发指南"}
{"file": ".trellis/spec/backend/api-module.md", "reason": "API 模块约定"}
{"file": ".trellis/spec/backend/quality.md", "reason": "代码质量要求"}
{"file": ".trellis/spec/frontend/index.md", "reason": "前端开发指南"}
{"file": ".trellis/spec/frontend/components.md", "reason": "组件约定"}
```

### 各 sub-agent 使用的 JSONL

| 文件              | Sub-agent           | 典型内容                                          |
| ----------------- | ------------------- | ------------------------------------------------- |
| `implement.jsonl` | `trellis-implement` | workflow + spec + 代码模式范例                    |
| `check.jsonl`     | `trellis-check`     | 质量 checklist spec + lint / typecheck 规则       |
| `research.jsonl`  | `trellis-research`  | 项目结构概览 + 调研目标                           |

### 0.5.0-beta.0 移除的内容

旧版本还引用过 `debug.jsonl` 和独立的 `debug` sub-agent。文件和 agent 都已移除，bug 修复在主会话或通过 `trellis-break-loop` 完成。

旧的 **completion marker** 机制（把 `reason` 字段转成 `TYPECHECK_FINISH` 这类字符串，供 Ralph Loop 判断 check agent 是否"完成"）也移除了。`trellis-check` sub-agent 自带的重试循环不再依赖这种 marker。
