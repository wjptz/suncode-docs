## 附录 C：`task.json` Schema 参考

```json
{
  "id": "string",           // 任务 ID，格式：MM-DD-slug
  "name": "string",         // Slug 名
  "title": "string",        // 任务标题
  "description": "string",  // 描述
  "status": "string",       // planning | in_progress | completed（默认三态）| rejected；扩展通过 workflow.md
  "dev_type": "string",     // backend | frontend | fullstack | test | docs
  "scope": "string",        // Commit scope，如 "auth"
  "priority": "string",     // P0 | P1 | P2 | P3（默认 P2）
  "creator": "string",      // 创建者
  "assignee": "string",     // 负责人
  "createdAt": "string",    // ISO 8601 创建时间
  "completedAt": "string",  // ISO 8601 完成时间（未完成时为 null）
  "branch": "string",       // 功能分支名
  "base_branch": "string",  // PR 目标分支
  "commit": "string",       // Commit hash
  "pr_url": "string",       // PR URL
  "parent": "string",       // 父任务目录名（顶层任务为 null）
  "children": [],           // 子任务目录名数组（父→子链接）
  "subtasks": [],           // 任务内部的 todo checklist（name + status 对），与 children 无关
  "relatedFiles": [],       // 相关文件列表
  "notes": "string",        // 备注
  "meta": {}                // 项目级元数据（如 Linear issue id）
}
```

### 父子关系 vs `subtasks`——别混

| 字段        | 含义                                                                                                    |
| ----------- | ------------------------------------------------------------------------------------------------------- |
| `parent`    | 父任务的目录名；顶层任务为 `null`。由 `task.py create --parent` 或 `add-subtask` 设置。                 |
| `children`  | 子任务目录名数组，与子任务的 `parent` 双向维护。                                                        |
| `subtasks`  | **单个任务内部的** `{name, status}` todo checklist。bootstrap 任务用到；和子任务关系无关。              |
