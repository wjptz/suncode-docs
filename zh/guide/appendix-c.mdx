## 附录 C：`task.json` Schema 参考

```json
{
  "id": "string",           // 任务 ID，格式：MM-DD-slug
  "name": "string",         // Slug 名
  "title": "string",        // 任务标题
  "description": "string",  // 描述
  "status": "string",       // planning | in_progress | review | completed | rejected
  "dev_type": "string",     // backend | frontend | fullstack | test | docs
  "scope": "string",        // Commit scope，如 "auth"
  "priority": "string",     // P0 | P1 | P2 | P3（默认 P2）
  "creator": "string",      // 创建者
  "assignee": "string",     // 负责人
  "createdAt": "string",    // ISO 8601 创建时间
  "completedAt": "string",  // ISO 8601 完成时间（未完成时为 null）
  "branch": "string",       // 功能分支名
  "base_branch": "string",  // PR 目标分支
  "commit": "string",       // Commit hash
  "pr_url": "string",       // PR URL
  "subtasks": [],           // 子任务列表
  "relatedFiles": [],       // 相关文件列表
  "notes": "string",        // 备注
  "meta": {}                // 项目级元数据（如 Linear issue id）
}
```

### 0.5.0-beta.0 移除的字段

| 字段              | 原因                                                              |
| ----------------- | ----------------------------------------------------------------- |
| `worktree_path`   | Multi-Agent Pipeline 移除                                         |
| `current_phase`   | 阶段流转由 sub-agent + skill 驱动，不再需要外部计数               |
| `next_action`     | 同上                                                              |

仍含这些字段的旧任务在 `task.py` 处理时保持原样，字段被忽略。
