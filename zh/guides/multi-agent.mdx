---
title: '并行会话'
description: '运行多个 AI 会话且不共享 active task 状态'
---

## 当前模型

Trellis 0.5 已移除旧的 Multi-Agent Pipeline、`/trellis:parallel`、`worktree.yaml`，以及 plan/dispatch/debug 那套 6-agent 流水线。现在直接使用 AI 工具自己的会话能力和 Git worktree。

关键变化是：active task 变成了**按会话隔离**：

```
.trellis/.runtime/sessions/<session-key>.json
```

这意味着窗口 A 可以做 task A，窗口 B 可以做 task B。一个 AI 会话里 start 任务，不会覆盖另一个会话的 active task。

## 推荐流程

1. 每个独立工作项创建一个 Trellis task。
2. 每个 task 打开一个 AI 会话/窗口。
3. 在每个会话里运行 `task.py start "$TASK_DIR"`。
4. 每个会话按正常 Plan → Execute → Finish 工作流推进。
5. 用你平时的 Git 流程 review 和 merge 分支。

需要代码隔离时，自己创建 Git worktree：

```bash
git worktree add ../feature-login -b feature/login
git worktree add ../feature-settings -b feature/settings
```

然后在每个 worktree 里分别打开 AI 会话。

## 适用场景

适合：

- 独立功能
- 并行修 bug
- 不同 package 或不同文件
- 每个分支都能独立 review 的实验

不适合：

- 多个任务会改同一批文件
- 一个任务依赖另一个任务未合并的改动
- 需要在一个对话里密集和用户来回确认的工作

## 上下文注入

Hook/extension-backed 平台会按当前会话注入 active task 上下文。Pull-based 平台由 sub-agent prelude 自己读取任务 JSONL。两种方式里，决定 active task 的都是 session key，不再有全局 active-task 指针。
