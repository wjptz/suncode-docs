## 任务是干什么的

任务是带上下文的工作项。每个任务知道：

- 你要做什么（PRD）
- 哪些文件相关
- 在哪个分支工作
- 当前状态

切换任务时，AI 的上下文也切换。不会搞混你在做哪个功能。

## 任务结构

任务在 `.trellis/tasks/`。每个任务是一个目录：

```
.trellis/tasks/01-31-user-auth-taosu/
├── task.json           # 元数据
├── prd.md              # 需求
├── implement.jsonl     # implement agent 的上下文
├── check.jsonl         # check agent 的上下文
└── debug.jsonl         # debug agent 的上下文
```

目录名格式：`MM-DD-简短描述-负责人`。

## 创建任务

手动创建任务目录：

```bash
mkdir -p .trellis/tasks/01-31-user-auth-taosu
```

然后在里面加上需要的文件。

### task.json

```json
{
  "id": "01-31-user-auth-taosu",
  "title": "添加用户认证",
  "status": "in_progress",
  "assignee": "taosu",
  "branch": "feature/user-auth",
  "created": "2026-01-31",
  "phases": ["implement", "check"]
}
```

### prd.md

写你要做什么。需求和验收标准要具体。

```markdown
# 用户认证

## 目标

用户可以用邮箱和密码注册、登录。

## 需求

- 用邮箱、密码、显示名注册
- 登录前需要验证邮箱
- 登录返回 JWT token
- Token 7 天过期
- 通过邮件重置密码

## API 接口

- POST /auth/signup
- POST /auth/login
- POST /auth/verify-email
- POST /auth/forgot-password
- POST /auth/reset-password

## 不在范围内

- 社交登录（Google、GitHub）
- 两步验证
```

### 上下文文件（JSONL）

告诉 agent 该读哪些文件。每行一个文件路径：

```jsonl
{"type": "spec", "path": ".trellis/spec/backend/index.md"}
{"type": "spec", "path": ".trellis/spec/backend/api.md"}
{"type": "file", "path": "src/routes/auth.ts"}
{"type": "file", "path": "src/services/user.ts"}
{"type": "file", "path": "src/db/models/user.ts"}
```

不同 agent 可以有不同的上下文文件。`implement.jsonl` 可能包含比 `check.jsonl` 更多的文件。

## 开始做任务

把任务 ID 写到 `.trellis/.current-task`：

```bash
echo "01-31-user-auth-taosu" > .trellis/.current-task
```

Hooks 用这个文件来注入正确的上下文。现在开 Claude Code 会话时，它知道：

- 读这个任务的 PRD
- 加载 `implement.jsonl` 里列的规范
- 在 `feature/user-auth` 分支工作

## 任务状态

直接编辑 `task.json` 的 `status` 字段：

```json
{
  "status": "in_progress"
}
```

常用状态：`pending`、`in_progress`、`review`、`done`。

## 多个任务

可以同时有多个进行中的任务。但同一时间只有一个是"当前"的。切换时更新 `.trellis/.current-task`：

```bash
echo "另一个任务id" > .trellis/.current-task
```

## 任务工作流

典型流程：

1. 创建任务，写需求
2. 设置上下文文件（哪些规范和代码相关）
3. 开始任务
4. 在 Claude Code 会话里工作
5. 完成后标记完成

任务系统不强制工作流。它只是把上下文组织好，让 AI 知道该关注什么。
