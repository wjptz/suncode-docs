## 附录 B：命令与 Skill 速查

Suncode 只保留很少的会话边界命令，其他都是 auto-trigger skill 或 sub-agent。这是有意的设计：命令负责用户显式边界，其他工作流自己跑。

### Slash 命令（按平台）

常规用户命令是 `finish-work` 和 `continue`。`start` 只在不会自动注入会话上下文的平台上作为用户入口出现。调用语法按平台不同：

| 平台                                                               | Start                                                                     | Finish                       | Continue                  | 交付形式                                |
| ------------------------------------------------------------------ | ------------------------------------------------------------------------- | ---------------------------- | ------------------------- | --------------------------------------- |
| Claude Code / OpenCode / Engineer / Gemini CLI / CodeBuddy / Droid | SessionStart hook 自动注入                                                | `/suncode:finish-work`       | `/suncode:continue`       | Slash 命令                              |
| Cursor / Pi Agent / Qoder                                          | SessionStart hook 或 extension 自动注入                                   | `/suncode-finish-work`       | `/suncode-continue`       | 原生命令 / prompt                       |
| Kiro                                                               | `@suncode:start`                                                          | `@suncode:finish-work`       | `@suncode:continue`       | Skill，通过 `@` 选择器                  |
| Kilo                                                               | `/start.md`                                                               | `/finish-work.md`            | `/continue.md`            | Workflow 文件（`.kilocode/workflows/`） |
| Antigravity / Devin                                                | workflow 选择器 → `start` / `finish-work` / `continue`                    |                              |                           | Workflow 文件                           |
| GitHub Copilot                                                     | Run Prompt → `suncode-start` / `suncode-finish-work` / `suncode-continue` |                              |                           | Prompt 文件（`.github/prompts/`）       |
| Codex                                                              | 自动读取 `AGENTS.md` prelude；可选 UserPromptSubmit hook                  | skill / prompt 入口          | skill / prompt 入口       | Skills + hooks                          |
| Oh My Pi / Grok Build                                              | 平台专用 Suncode 命令                                                     | Suncode 命令                 | Suncode 命令              | 原生命令文件                            |
| Kimi Code                                                          | `/skill:suncode-start`                                                    | `/skill:suncode-finish-work` | `/skill:suncode-continue` | Kimi 私有 skill                         |

### Auto-trigger Skill

由平台根据用户意图自动匹配，不需要显式调用。需要时用各平台的 skill 调用语法手动触发。

| Skill                 | 触发时机                                 | 干什么                                           |
| --------------------- | ---------------------------------------- | ------------------------------------------------ |
| `suncode-brainstorm`  | 用户描述功能 / bug / 模糊需求            | 产出 task + `prd.md`，必要时 spawn research      |
| `suncode-before-dev`  | task 进入 in_progress、AI 即将动手改代码 | 读该 package 相关的 spec                         |
| `suncode-check`       | 实现阶段结束                             | diff 审查 + lint / typecheck / test + 自修复循环 |
| `suncode-update-spec` | 学到 / 决策 / 踩到值得沉淀的东西         | 在对应 spec 文件添加条目                         |
| `suncode-break-loop`  | 刚修完棘手 bug                           | 5 维根因 + 预防机制分析                          |

### Sub-agent

主会话通过平台的 sub-agent / Task 原语 spawn。

| Sub-agent           | 角色          | 约束            |
| ------------------- | ------------- | --------------- |
| `suncode-research`  | 代码/文档搜索 | 只读            |
| `suncode-implement` | 编码          | 不 `git commit` |
| `suncode-check`     | 验证 + 自修复 | 自带重试循环    |

Sub-agent 工作流在 Claude Code、Cursor、OpenCode、Engineer、Codex、Kiro、Gemini CLI、Qoder、CodeBuddy、Copilot、Droid、Pi Agent、Oh My Pi、Reasonix、ZCode、Trae、Grok Build 和 Kimi Code 可用。Kilo、Antigravity、Devin 在主会话内联运行。Codex 启用 dispatch 时走原生 `SubagentStart` 且保留 pull fallback；Grok 和 Kimi 走 pull-based。

### 全局 CLI 命令

| 命令                    | 用途                                  | 用法                                                               |
| ----------------------- | ------------------------------------- | ------------------------------------------------------------------ |
| `suncode upgrade`       | 升级全局安装的 Suncode CLI package    | `suncode upgrade [--tag <tag>]`                                    |
| `suncode update`        | 把当前项目的 Suncode 文件同步到此 CLI | `suncode update [--dry-run] [--migrate] [...]`                     |
| `suncode platforms`     | 报告已配置的 Suncode 平台             | `suncode platforms [--json]`                                       |
| `suncode channel spawn` | 注册协作 worker；Codex 可覆盖 sandbox | `suncode channel spawn <name> --provider codex [--sandbox <mode>]` |

`suncode platforms --json` 输出稳定的
`{"platforms":[{"id":"codex","displayName":"Codex","configDir":".codex"}]}`；空结果是
`{"platforms":[]}` 且 exit 0。Codex channel sandbox 可选 `read-only`、
`workspace-write`（默认）或 `danger-full-access`；其他 provider 忽略此选项。

### Hub CLI 命令

Hub 命令是可选能力。只有 `.suncode/config.yaml` 包含 `hub.enabled: true`，并且当前项目已通过 `suncode hub login` 登录时，才进入团队协作路径。

| 命令                            | 用途                                                           | 用法                                                               |
| ------------------------------- | -------------------------------------------------------------- | ------------------------------------------------------------------ |
| `suncode hub init`              | 写入 Hub 项目配置和用户级默认 Hub URL                          | `suncode hub init [--no-auto-pull-spec] [--yes] [...]`             |
| `suncode hub login`             | 登录配置的 Hub 服务                                            | `suncode hub login [--api-base-url <url>] [--email <email>]`       |
| `suncode hub logout`            | 删除某个服务的本地 Hub session                                 | `suncode hub logout [--api-base-url <url>]`                        |
| `suncode hub state`             | 查看配置、登录、服务、工作项和当前任务状态                     | `suncode hub state [--json] [--prompt] [--hook]`                   |
| `suncode hub status`            | 查看当前项目是否启用 Hub                                       | `suncode hub status`                                               |
| `suncode hub pull`              | 列出分配给当前开发者的 requirements                            | `suncode hub pull`                                                 |
| `suncode hub intake`            | 把 Hub requirement 接取成本地 Suncode task                     | `suncode hub intake --auto` 或 `--requirement <id>`                |
| `suncode hub pull-spec`         | 拉取 Hub 里的权威项目 spec bundle                              | `suncode hub pull-spec [--json]`                                   |
| `suncode hub sync`              | 拉取最新 requirement 详情和需求变更                            | `suncode hub sync --task current [--cursor <cursor>]`              |
| `suncode hub plan-ready`        | 提交 plan artifacts 和结构化 subtasks                          | `suncode hub plan-ready --task current [--debug]`                  |
| `suncode hub review`            | 运行实施后的 Hub code review 轮次                              | `suncode hub review --task current [--module <path>]`              |
| `suncode hub latest-review`     | 读取最新本地 Hub review 结果                                   | `suncode hub latest-review --task current`                         |
| `suncode hub pull-review`       | 拉取 Hub 侧 plan/start review 评论                             | `suncode hub pull-review --task current [--cursor <cursor>]`       |
| `suncode hub finish`            | 提交最终 spec 和 completion artifacts                          | `suncode hub finish --task current [--file <path>]`                |
| `suncode hub submit-plan`       | 上传当前任务规划 artifacts                                     | `suncode hub submit-plan --task current [--force]`                 |
| `suncode hub submit-subtasks`   | 上传从 `subtasks.json` 或 `implement.md` 得到的结构化 subtasks | `suncode hub submit-subtasks --task current [--force]`             |
| `suncode hub submit-spec`       | 上传项目级 spec artifacts                                      | `suncode hub submit-spec --task current [--file <path>] [--force]` |
| `suncode hub submit-completion` | 上传 completion artifacts                                      | `suncode hub submit-completion --task current [--force]`           |
| `suncode hub create-task`       | 为本地任务创建或绑定远端 Hub task                              | `suncode hub create-task --task current [--best-effort]`           |
| `suncode hub preflight-start`   | 询问 Hub 当前任务是否可以开始开发                              | `suncode hub preflight-start --task current`                       |
| `suncode hub mark-started`      | 标记已绑定 Hub task 开始开发                                   | `suncode hub mark-started --task current [--status in_progress]`   |
| `suncode hub sync-pending`      | 重试排队的 best-effort Hub 生命周期同步失败                    | `suncode hub sync-pending`                                         |
| `suncode hub knowledge`         | 搜索当前项目 Hub 知识库                                        | `suncode hub knowledge "<query>" [--top-k <n>]`                    |
| `suncode hub skill-push`        | 上传本地 `.agents/skills/<name>/` package                      | `suncode hub skill-push <skill-name>`                              |
| `suncode hub skill-pull`        | 下载 Hub skill package 到 `.agents/skills/`                    | `suncode hub skill-pull <skill-name>`                              |
| `suncode hub agent-push`        | 上传本地 `.suncode/agents/<name>.md` package                   | `suncode hub agent-push <agent-name>`                              |
| `suncode hub agent-pull`        | 下载 Hub agent package 到 `.suncode/agents/`                   | `suncode hub agent-pull <agent-name>`                              |

`suncode hub spec-deletions list|keep|discard` 用来处理 Hub 删除但 Suncode 本地保留下来的 spec 候选。`suncode hub download-document` 是处理 Hub signed document payload 的底层辅助命令。

### `task.py` 子命令

| 子命令            | 用途                                                                       | 用法                                                               |
| ----------------- | -------------------------------------------------------------------------- | ------------------------------------------------------------------ |
| `create`          | 创建任务（检测到 sub-agent 平台时 seed `implement.jsonl` / `check.jsonl`） | `task.py create "title" [--slug name] [-a assignee] [-p priority]` |
| `add-context`     | 追加上下文条目（create 之后填 jsonl 的主要入口）                           | `task.py add-context "$DIR" <target> "<path>" "<reason>"`          |
| `validate`        | 校验 JSONL                                                                 | `task.py validate "$DIR"`                                          |
| `list-context`    | 查看所有条目                                                               | `task.py list-context "$DIR"`                                      |
| `start`           | 置为当前 AI 会话/窗口的任务                                                | `task.py start "$DIR"`                                             |
| `current`         | 查看当前 AI 会话/窗口的任务                                                | `task.py current [--source] [--json]`                              |
| `finish`          | 清除当前 AI 会话/窗口的任务                                                | `task.py finish`                                                   |
| `set-branch`      | 设置分支名                                                                 | `task.py set-branch "$DIR" "feature/xxx"`                          |
| `set-base-branch` | 设置 PR 目标分支                                                           | `task.py set-base-branch "$DIR" "main"`                            |
| `set-scope`       | 设置范围                                                                   | `task.py set-scope "$DIR" "auth"`                                  |
| `add-subtask`     | 链接子任务到父任务                                                         | `task.py add-subtask <父任务目录> <子任务目录>`                    |
| `remove-subtask`  | 解除子任务关联                                                             | `task.py remove-subtask <父任务目录> <子任务目录>`                 |
| `archive`         | 归档任务                                                                   | `task.py archive <name>`                                           |
| `list`            | 列出活跃任务                                                               | `task.py list [--mine] [--status <s>] [--json]`                    |
| `list-archive`    | 列出归档任务                                                               | `task.py list-archive [YYYY-MM]`                                   |

`task.py create` 还接受 Hub 元数据选项，例如 `--hub-project-id`、`--hub-developer-id`、`--hub-requirement-id`、`--hub-requirement-revision`、`--hub-task-role`、`--hub-parent-local-task-id`、`--hub-parent-remote-task-id`。常规接取 Hub 需求仍应使用 `suncode hub intake`；这些参数主要用于高级绑定和测试。

`task.py current --json` 输出 `{current_task, source, stale}`，
`current_task` 为 `null` 时 exit 1。`task.py list --json` 输出
`{tasks:[...]}`，每项同时有持久化 `status` 和派生 `display_status`；
因此父任务可显示为 `active`，但磁盘上仍保持 `planning`。

### Python 脚本

```bash
# 上下文
./.suncode/scripts/get_context.py                 # 完整上下文
./.suncode/scripts/get_context.py --json          # JSON
./.suncode/scripts/get_context.py --mode packages # 按 package 的 spec 层（monorepo）
./.suncode/scripts/get_context.py --mode record   # 给 /suncode:finish-work 用

# 会话
./.suncode/scripts/add_session.py --title "..." --commit "..." --summary "..."

# Spec bootstrap（首次）
./.suncode/scripts/create_bootstrap.py
```
