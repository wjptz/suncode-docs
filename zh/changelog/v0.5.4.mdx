| Workflow 协议 | `packages/cli/src/templates/trellis/workflow.md`                                                                                                 | `[workflow-state:in_progress]` 把 dispatch 规则限定在主会话；增加 sub-agent 自豁免说明。Phase 2 dispatch 段加一行 `Dispatch prompt guard`，要求 spawn 时告诉子 agent "你已经是被 dispatch 的那个 agent"。                                                                                               |

### `compareVersions`：prerelease 含连字符时保留尾部

感谢 [@voidborne-d](https://github.com/voidborne-d) 的 [`#230`](https://github.com/mindfold-ai/Trellis/pull/230)。

`packages/cli/src/utils/compare-versions.ts` 是 Trellis 共享的 SemVer 比较器，CLI 启动时检测项目版本、migration 排序/过滤、npm 更新检查都依赖它。原实现用 `a.split("-", 2)` 切分 base 和 prerelease——但 JS 的 `String.prototype.split("-", 2)` 限制参数是**截尾**而不是合并尾部（跟 Python `maxsplit` 不一样）：

```js
'1.0.0-alpha-1'.split('-', 2);
// → ["1.0.0", "alpha"]   // "-1" 被静默丢掉
```

SemVer §9 允许 prerelease 标识里含连字符（`1.0.0-alpha-1`、`1.0.0-x-y-z`），所以仅在第二个连字符之后才有差异的版本之前会被判等：

| 输入对                             | 旧结果 | 正确 |
| ---------------------------------- | ------ | ---- |
| `1.0.0-alpha-1` vs `1.0.0-alpha-2` | `0`    | `-1` |
| `1.0.0-rc-foo` vs `1.0.0-rc-bar`   | `0`    | `1`  |
| `1.0.0-x-y-z` vs `1.0.0-x-y`       | `0`    | `1`  |

改用 `indexOf("-")` + `slice`，第一个连字符之后的全部内容作为 prerelease 字符串保留。新增 `packages/cli/test/utils/compare-versions.test.ts`（20 个 case）覆盖基础版本、release vs prerelease、prerelease 标识、含连字符标识、`Array.prototype.sort` 比较器契约（`getMigrationsForVersion` / `getAllMigrationVersions` 依赖此契约）。
