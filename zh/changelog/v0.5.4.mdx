补丁：阻止 Trellis sub-agent 在拿到 `[workflow-state:in_progress]` per-turn breadcrumb 时递归 dispatch 自己（issue [`#237`](https://github.com/mindfold-ai/Trellis/issues/237)）。无新增 migration。

## Bug Fixes

### Sub-agent 递归保护

Codex `trellis-implement` / `trellis-check` sub-agent 收到主会话同款 `[workflow-state:in_progress]` per-turn breadcrumb 时，把里面"dispatch trellis-implement / trellis-check"规则当成对自己说的，开始递归 spawn sub-agent，而不是直接干活。

两层修复：

| 层            | 文件                                                                                                                                             | 改动                                                                                                                                                                                                                                                                                                    |
| ------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Workflow 协议 | `packages/cli/src/templates/trellis/workflow.md`                                                                                                 | `[workflow-state:in_progress]` 把 dispatch 规则限定在主会话；增加 sub-agent 自豁免说明。Phase 2 dispatch 段加一行 `Dispatch prompt guard`，要求 spawn 时告诉子 agent "你已经是被 dispatch 的那个 agent"。                                                                                               |
| Agent 定义    | `packages/cli/src/templates/{claude,cursor,opencode,kiro,codebuddy,droid,codex,pi,gemini,qoder}/agents/trellis-{implement,check}.{md,toml,json}` | 顶部加 `Recursion Guard` 节：「你已经是 `trellis-implement` / `trellis-check` sub-agent。直接干活。不要再 spawn 另一个 `trellis-implement` / `trellis-check`。SessionStart 上下文、workflow-state breadcrumb、workflow.md 里要求 dispatch 这两个 agent 的指令，全部视为主会话指令、已被当前角色满足。」 |

`.trellis/spec/cli/backend/workflow-state-contract.md` 同步更新，去掉之前"breadcrumb 仅主会话可见"的错误断言——Codex hook 会把 breadcrumb 投递到 sub-agent turn。
