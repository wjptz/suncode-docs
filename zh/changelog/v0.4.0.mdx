**v0.4.0 正式版。** 从 rc.1 直接 promote，`src/` 零改动——这是 0.4.0 周期 11 个 beta + 2 个 RC 全部内容的稳定版。

## 主线：Trellis 原生支持 Monorepo

0.4.0 的定性改动是结构性的：**Trellis 现在以「包」为单位思考，而不是「项目」**。由两个紧耦合的重构共同完成：

### 1. Monorepo 支持 + per-package spec 基础设施

`trellis init` 会识别 monorepo，按包创建**独立的** spec 目录，而不是项目全局一份。所有操作 task 状态的 Python 脚本——`task.py`、`add_session.py`、`create_bootstrap.py`——都加了 `--package` 参数。`start` / `parallel` / `workflow` / `agents` 模板里的 spec 发现逻辑也按 package 作用域走。

这改变了核心心智模型——从"一个项目一套 spec"变成"一个项目 N 套 spec，按包切分"。

### 2. 通用 `before-dev` / `check` 命令

如果只做 per-package spec，命令矩阵会爆炸成 `N 个包 × {backend, frontend}`，根本用不起来。所以在 9 个平台上，把类型特定的 `before-backend-dev` / `before-frontend-dev` / `check-backend` / `check-frontend` 全部合并成单一的 `before-dev` / `check`。

为了安全落地，新增了一种迁移类型——**`safe-file-delete`**：只删除 hash 匹配已知未修改状态的文件，你的本地改动会被保留。beta.1 的 manifest 里塞了 36 条 `safe-file-delete`，覆盖所有平台的所有变体。

**为什么这两个是一个改动：** 没有命令合并，per-package 架构实操上不可用；没有 per-package，命令合并只是清理。绑在一起才构成 0.4.0 的架构代差。

## 生态：Shared Agent Skills 共享层

Codex 现在会把 skill 写入 `.agents/skills/`——这是 [agentskills.io](https://agentskills.io) 标准的跨客户端目录。Cursor、Gemini CLI、GitHub Copilot、Amp、Kimi Code 都会自动读取。`trellis init` 的 Codex 选项把这个「顺带覆盖多个工具」的好处显式标出来，让用户知道勾一个选项能覆盖多个工具链。

## 新增平台

0.4.0 周期接入了 5 个新平台，都是在 0.3.x 抽象之上的 additive 扩展：

- **GitHub Copilot** —— 完整平台支持，包含 hooks 追踪和独立 prompt 模板
- **Windsurf** —— workflow 支持
- **Qoder** —— skills 类型平台，CLI 为 `qodercli`
- **Kimi Code** —— 通过共享层自动读取 `.agents/skills/`
- **Factory Droid**（`--droid`）—— Cursor 级别支持，仅 commands（`.factory/commands/trellis/<name>.md`）

## 值得一提的改进

- **自托管模板 Registry。** `trellis init` / `trellis update` 支持通过 `TRELLIS_REGISTRY` 环境变量从自托管 URL 拉取模板。可用于企业内部镜像和离线环境部署。
- **多 Agent worktree + PR 感知。** 多 agent 流水线能追踪 worktree submodule 和 PR 状态。
- **重新 init 的快速路径。** 在已有项目里跑 `trellis init --codex` 只配置 Codex；`trellis init -u name` 为新设备设置开发者身份；不带参数的 `trellis init` 会显示 add-platform / add-developer / 完整 re-init 菜单。
- **分支感知的 session 和 journal。** `.trellis` 的 session 和 journal 记录带上 git 分支上下文。
- **Statusline 集成。** Claude Code 状态栏显示当前 Trellis 任务状态。
- **Marketplace 拆为独立仓库。** marketplace 迁移为 git submodule（#117）。
- **Python 脚本重构。** 引入共享的 `io` / `log` / `git` 模块、`TaskInfo` dataclass，把 god module 拆成职责清晰的小模块。
- **OpenCode 插件 v1。** 升级到 v1 插件 API，修复 context 注入不持久化的问题（之前会话重开 context 会丢）。感谢 @Adamcf123。

## Bug 修复

**SessionStart additionalContext 瘦身。** vanilla 从约 29 KB 缩到约 7 KB，控制在 Claude Code 约 20 KB 的截断阈值之下。在 rc.0 之前，大多数项目的 task 状态会被静默丢失——这可能是整个周期里影响面最大的一个修复。

**Windows 修复。**
- `statusline.py` 在 Windows 上因 GBK 编码崩溃（中点分隔符），现在把 stdout/stderr 包成 UTF-8。感谢 @xiangagou163。
- Codex `hooks.json` 里的 `{{PYTHON_CMD}}` 占位符解析（#132）
- `isManagedPath` 路径规范化（`\` → `/`）

**其他修复（节选）。**
- fix(update): rename 迁移允许指向受保护路径 + 配置解析失败时告警
- fix(update): 创建迁移任务时从 `.developer` 解析名字
- fix(hooks): 跨平台规范化 `.current-task` 路径引用（#130）
- fix(hooks): ralph-loop 的 `SubagentStop` 事件字段名（#152）
- fix(opencode): dispatch 等待子任务（#147）
- fix(init): monorepo 包目录去掉 npm scope 前缀
- fix(init): 模板选择器把 "empty templates" 改成 "from scratch"
- fix(scripts): `start.py` 保留 submodule 的 status 前缀

## 升级说明

- **0.3.x 升级：** 跑 `trellis update` 会应用整个 0.4.0 周期。被合并的 `before-dev` / `check` 命令对应的 36 个文件删除，会通过 `safe-file-delete` 自动处理（带 hash 校验，本地改动会保留）。
- **任意 0.4.0 prerelease 升级：** `trellis update` 是一次干净的版本升级，没有迁移。
