## Enhancements

- **`tl mem extract <id> --phase brainstorm`** —— 把一个 session 里的讨论部分（`task.py create` 到 `task.py start` 之间）单独切出来。多 task session 会用 `--- task: <slug> ---` 分隔。`--phase implement` 反向输出实施部分；`--phase all` 是默认完整输出。

  ```bash
  tl mem extract <id> --phase brainstorm
  tl mem extract <id> --phase brainstorm --json
  tl mem extract <id> --phase implement
  ```

  Claude 和 Codex 支持。OpenCode 暂回退到完整对话。

- **`tl mem` 整体快了 5-9 倍。**

  | 命令                             | 修前 | 修后  |
  | -------------------------------- | ---- | ----- |
  | `mem list`                       | 3.5s | 0.67s |
  | `mem list --platform codex`      | 3.2s | 0.33s |
  | `mem extract --phase brainstorm` | 5.8s | 0.73s |

## Bug Fixes

- **OpenCode 1.2+ 用户的 `tl mem` 不再返回 0 sessions**。OpenCode 1.2 把 session 存到 SQLite，旧实现还在读已经空的 JSON 目录，装了新版 OpenCode 的人完全用不了 `tl mem`。修了。OpenCode 1.1.x 不再支持。

- **`--phase` 解析器认得 `$(... --slug NAME)` 命令替换、单条 Bash 里的多个 `task.py` 调用、commit message heredoc 里字面引用的 `task.py start`**。
