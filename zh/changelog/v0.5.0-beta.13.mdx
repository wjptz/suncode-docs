beta.12 的补丁跟进。`task.py start` 现在会把 `task.json` 的 status 从 `planning` 真正切到 `in_progress`，之前只改了 `.current-task` 指针，status 字段未修复，导致 cc 的 status line 会一直显示 `planning`。

## Bug Fixes

### `task.py start` 会把 `task.json` 状态切到 `in_progress`

`cmd_start` 之前就写个 `.current-task`、跑下 `after_start` hook 完事。`status` 字段只在 create 时写 `planning`、archive 时写 `completed`，中间没人改。所以一个 task 干了一周，`list --status in_progress` 还是空的，session-start 面包屑也一直是 `Status: PLANNING`。

现在 `cmd_start` 设完指针后读一下 `task.json`，只有 `planning` 才翻成 `in_progress`，其他状态不动：

| 当前 status | `task.py start` 之后 |
|---|---|
| `planning` | `in_progress` |
| `in_progress` | `in_progress`（幂等） |
| `review` | `review`（保留，review 返工再 start 不能清回退） |
| `completed` | `completed`（保留） |

位置 `packages/cli/src/templates/trellis/scripts/task.py:cmd_start`。

### Codex agent 模板回填 Phase 1.3 fallback 段

beta.11 做 init-context-removal 的时候，在 Codex 两个 agent 文件顶上加了一段 "Required: Load Trellis Context First"，告诉 sub-agent 跳过 `{"_example": ...}` seed 行、jsonl 没 curated 条目就退回去看 prd.md + 用 `get_context.py --mode packages` 自己挑 spec。

但这段只改了项目根下的 dogfood 副本，没回传到 `packages/cli/src/templates/codex/agents/`。到 beta.12 为止，Codex 用户新装或 update 下来的 agent prompt 是缺这段的，遇到 seed-only jsonl 就直接卡住，不会退回 prd.md。

这版补回了：

- `packages/cli/src/templates/codex/agents/trellis-check.toml`
- `packages/cli/src/templates/codex/agents/trellis-implement.toml`

和 `configurators/shared.ts:buildPullBasedPrelude` 给 Copilot / Gemini / Qoder 生成的 prelude 一模一样，Codex 行为现在和其他 pull-based 平台追齐了。

## Docs

### `.codex/config.toml` 文档化 `features.codex_hooks` 开关

Trellis 的 Codex 集成靠 `.codex/hooks.json` 里的 SessionStart + UserPromptSubmit 两个 hook。但 Codex 只认 **user-level** 的 `~/.codex/config.toml` 里的 `[features] codex_hooks = true`，项目级 `.codex/config.toml` 写 `features.*` 是不生效的。没开这个 flag，`hooks.json` 就被直接忽略，hook 不跑，看起来像 Trellis 坏了。

现在项目级 `.codex/config.toml` 改成注释，直接告诉用户去 `~/.codex/config.toml` 加什么。顺手删了老的 `shell_environment_policy` 段（和 Trellis 无关的死配置）。

## Upgrade

已有项目：

```bash
trellis update
```

没迁移步骤。旧 task、jsonl、`.current-task` 都照旧。升级完对一个还在 `planning` 的旧 task 第一次跑 `task.py start`，status 字段会自动翻成 `in_progress`。手改过 `.codex/agents/trellis-*.toml` 的用户会在 update 时被问"这俩文件变了要不要覆盖"，按提示选就行。

全新安装：

```bash
npm i -g @mindfoldhq/trellis@0.5.0-beta.13
```
