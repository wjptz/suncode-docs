任务生命周期 hooks、自定义模板仓库、父子 subtask，以及一个关键的 hook 修复。

## 新增功能

- **自定义模板仓库**（`--registry`）：在 `trellis init` 时从你自己的 GitHub、GitLab 或 Bitbucket 仓库拉取 Spec 模板。支持 marketplace 模式（含 `index.json`）和直接下载模式。详见[远程 Spec 模板](/zh/guide/ch02-quick-start#25-远程-spec-模板)。
- **任务生命周期 hooks**：在 `.trellis/config.yaml` 中配置 `after_create`、`after_start`、`after_finish`、`after_archive` 四个事件的 hook。每个 hook 通过 `TASK_JSON_PATH` 环境变量接收任务信息，附带 Linear sync hook 示例。详见[任务生命周期 Hooks](/zh/guide/ch06-task-management#66-任务生命周期-hooks)。
- **父子 subtask 支持**：新增 `add-subtask` 和 `remove-subtask` 命令，`task.py create` 支持 `--parent` 参数，task.json 新增 `children`/`parent`/`meta` 字段。
- **record-session 改进**：归档判断现在基于实际工作状态，而非 task.json 的 `status` 字段。
- **brainstorm subtask 拆解**：`/trellis:brainstorm` 第 8 步新增针对复杂任务的子任务创建流程。

## Bug 修复

- **PreToolUse context 注入被 Claude Code v2.1.63 破坏**：CC 将内部工具从 `Task` 改名为 `Agent`（[anthropics/claude-code#29677](https://github.com/anthropics/claude-code/issues/29677)），导致 PreToolUse hook 静默失效——`implement`、`check`、`debug`、`research` agent 均未收到注入的 code-spec 上下文。修复方式：同时接受 `Task` 和 `Agent` 两种工具名。

<Warning>
  PreToolUse bug 影响**所有**使用 Claude Code v2.1.63+ 的 Trellis 用户。请运行 `trellis update`
  应用修复。
</Warning>

<Note>运行 `trellis update` 以同步更新的 hook 脚本和命令模板。</Note>
