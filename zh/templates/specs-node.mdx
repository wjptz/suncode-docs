## 项目结构

```
src/
├── routes/     # Express 路由处理器
├── services/   # 业务逻辑
├── db/
│   ├── models/     # 数据库模型
│   └── migrations/ # SQL 迁移
├── lib/        # 共享工具
└── types/      # TypeScript 类型
```

## 错误处理

所有错误用 `src/lib/errors.ts` 的 `AppError`。不要 throw 原生 `Error` 对象。

API 错误响应格式：

```json
{
  "error": {
    "code": "USER_NOT_FOUND",
    "message": "用户不存在"
  }
}
```

生产环境不暴露堆栈。

## 数据库

- 表名：`snake_case`，复数（`user_sessions`）
- 列名：`snake_case`（`created_at`）
- 必须有 `id`、`created_at`、`updated_at`
- 外键：`{单数表名}_id`（`user_id`）
- 用 `src/db/query.ts` 的查询构建器，不写原生 SQL

## API 路由

路由放 `src/routes/`。每个资源一个文件。

```typescript
// src/routes/users.ts
import { Router } from 'express';
import { UserService } from '../services/user';

const router = Router();

router.get('/:id', async (req, res) => {
  const user = await UserService.findById(req.params.id);
  res.json(user);
});

export default router;
```

用 Zod 验证输入。Schema 放 `src/types/schemas.ts`。

## 认证

JWT token 做认证。Token 处理在 `src/lib/auth.ts`。

受保护的路由用 `requireAuth` 中间件：

```typescript
router.get('/profile', requireAuth, async (req, res) => {
  // req.user 在这里可用
});
```
