## 后端模板

### Node.js + Express + TypeScript

<CodeGroup>
```markdown 后端规范
# 后端规范

## 项目结构

src/
├── routes/ # Express 路由处理器
├── services/ # 业务逻辑
├── db/
│ ├── models/ # 数据库模型
│ └── migrations/ # SQL 迁移
├── lib/ # 共享工具
└── types/ # TypeScript 类型

## 错误处理

所有错误用 src/lib/errors.ts 的 AppError：

throw new AppError("USER_NOT_FOUND", "用户不存在", 404);

API 错误响应格式：

{ "error": { "code": "USER_NOT_FOUND", "message": "用户不存在" } }

不要 throw 原生 Error 对象。生产环境不暴露堆栈。

## 数据库

- 表名：snake_case，复数（user_sessions）
- 列名：snake_case（created_at）
- 必须有 id、created_at、updated_at
- 外键：{单数表名}\_id（user_id）

用 src/db/query.ts 的查询构建器，不写原生 SQL。

## API 路由

路由放 src/routes/。每个资源一个文件。

用 Zod 验证输入。Schema 放 src/types/schemas.ts。

## 认证

JWT token 做认证。Token 处理在 src/lib/auth.ts。

受保护的路由用 requireAuth 中间件。

````
</CodeGroup>

### Python + FastAPI

<CodeGroup>
```markdown FastAPI 规范
# 后端规范

## 项目结构

app/
├── api/
│   └── routes/     # FastAPI 路由处理器
├── services/       # 业务逻辑
├── db/
│   ├── models/     # SQLAlchemy 模型
│   └── migrations/ # Alembic 迁移
├── core/           # 配置、安全、依赖
└── schemas/        # Pydantic schemas

## 错误处理

自定义异常在 app/core/exceptions.py。
用全局异常处理器。不要 catch 再 re-raise。

## 数据库

- 表名：snake_case，复数
- 用 SQLAlchemy ORM，不写原生 SQL
- 模型在 app/db/models/
- 每个模型有 id、created_at、updated_at

## API 路由

每个资源一个 router 在 app/api/routes/。

## 验证

所有输入输出用 Pydantic schema。
````

</CodeGroup>

## 前端模板

### React + TypeScript

<CodeGroup>
```markdown 前端规范
# 前端规范

## 项目结构

src/
├── components/ # 复用组件
├── pages/ # 路由页面
├── hooks/ # 自定义 hooks
├── stores/ # Zustand stores
├── lib/ # 工具
├── types/ # TypeScript 类型
└── api/ # API 客户端函数

## 组件

只用命名导出。不用默认导出。

Props 接口定义在组件上面。不用 inline 类型。

## 状态管理

用 Zustand。Store 文件在 src/stores/。

## API 调用

API 函数在 src/api/。用 src/lib/fetch.ts 的 fetch 封装。

## 样式

Tailwind CSS。不用 inline styles。不用 CSS modules。

````
</CodeGroup>

## 全栈模板

### Next.js + tRPC + Prisma

<CodeGroup>
```markdown 全栈规范
# 全栈规范

## 项目结构

src/
├── app/            # Next.js app router
├── server/
│   ├── api/        # tRPC routers
│   └── db/         # Prisma client
├── components/     # React 组件
├── hooks/          # 自定义 hooks
└── lib/            # 共享工具
prisma/
└── schema.prisma   # 数据库 schema

## API (tRPC)

Routers 在 src/server/api/routers/。

## 数据库 (Prisma)

模型在 prisma/schema.prisma。
schema 改了后跑 npx prisma generate。

## 组件

跟 React 规范一样。用 tRPC hooks。
````

</CodeGroup>

## 使用这些模板

1. 复制模板到 `.trellis/spec/backend/index.md` 或 `.trellis/spec/frontend/index.md`
2. 调整路径匹配你的项目
3. 加上项目特定的规范
4. 删掉不适用的部分

模板是起点。你的规范应该引用你代码库里的实际文件。
