Suncode 默认可以完全本地运行。团队 Hub 是一层协作能力，用来把共享需求、任务归属、评审状态和 agent 行为提示连接起来，让多人和多个 agent 面对同一份工作状态。

<Note>Suncode Hub 是可选能力。如果当前项目没有接入 Hub，继续使用普通本地 Suncode 工作流即可。</Note>

## 什么时候用 Hub

<CardGroup cols={2}>
  <Card title="使用 Hub" icon="users">
    团队有共享需求队列，并希望本地 Suncode task 和团队工作项保持绑定。
  </Card>
  <Card title="保持本地" icon="laptop">
    你在做私人实验、一次性维护，或不需要进入团队协作视图的工作。
  </Card>
</CardGroup>

当不止一个人或 agent 需要看到同一份工作状态时，Hub 最有价值。个人任务、快速修复和项目内探索仍然更适合直接走本地 Suncode。

## 设置 Hub

在项目根目录运行：

```bash
suncode hub init
suncode hub login
suncode hub state
```

`suncode hub init` 会引导你填写团队提供的 Hub 位置和项目 ID。`suncode hub login` 会让这台机器登录团队工作区。`suncode hub state` 用来确认当前项目是否已配置、已登录、可连接，并且是否可以开始处理 Hub 工作。

<Tip>开始团队任务前先运行 `suncode hub state`。这是判断该走 Hub 流程还是本地流程的最快方式。</Tip>

## 默认 Hub 流程

配置完成后，日常使用这几个高层命令：

```bash
suncode hub state
suncode hub intake --auto
suncode hub plan-ready --task current
suncode hub review --task current
suncode hub finish --task current
```

`hub intake --auto` 只有在恰好一个可接需求时才会接取。多个需求可接时会返回 ambiguous；先选择需求 ID，再运行 `suncode hub intake --requirement <id>`。自动创建的 Hub task 会让 `task.json.name` 和 `task.json.title` 以 `HUB-REQ-<requirementId>` 开头；中文标题保留在这个前缀之后，目录 slug 保持 ASCII。

`hub plan-ready` 会提交规划 artifacts，从 `implement.md` 派生结构化 subtasks（有 `subtasks.json` 时使用 override），上传 subtasks，并运行 Hub start preflight。`task.py start` 也会在本地状态改变前对 Hub-bound task 运行 blocking preflight gate。

## Review Hub 绑定任务

项目可以启用以 task 为中心的 review 轮次，不需要先启用完整的分支/合并模式：

```bash
suncode hub review --task current
```

当 `hub.review.enabled` 为 `true` 时，Suncode 会创建下一轮本地 review，采集当前 diff，调用配置的 provider，写入 `reviews/round-NNN/review.json` 和简洁的 `result.md` 报告，通过 Hub 上传 review artifacts，并同步 task 状态。第一版支持的 provider 是 Engineer，对应配置是 `hub.review.provider: engineer`。只有 `hub.review.engineer.saveRawOutput` 为 true 时，provider 原始执行输出才会保存在 `raw-output.md`。

默认 `hub.review.required` 是 `false`：review 结果会同步到 Hub，但不会阻塞 `suncode hub finish`。如果团队要求完成前必须审核通过，可以设为 `hub.review.required: true`；此时 completion 会要求最新 approved review 仍然匹配当前 diff。

## Agent 如何使用 Hub 状态

支持 Suncode 的 agent 会在会话中看到一段紧凑的 Hub 状态摘要。Agent 用它来选择安全路径：

| Hub 状态           | Agent 行为                                    |
| ------------------ | --------------------------------------------- |
| Hub 关闭或未配置   | 使用普通本地 Suncode 工作流。                 |
| 需要登录           | 提醒你先运行 `suncode hub login`。            |
| Hub 当前不可连接   | 不启动 Hub 专用动作；继续本地工作或稍后重试。 |
| 有可处理团队工作   | 使用 `hub intake`；多个需求时不要替用户猜。   |
| 当前任务是本地任务 | 不对这个任务运行 Hub 提交或状态同步动作。     |
| 当前任务已绑定 Hub | 让本地进度和团队 Hub 工作项保持一致。         |

这条护栏很重要：普通本地任务和 Hub 绑定任务在文件树里可能很像。Hub 状态摘要能让 agent 在同步任何内容之前先判断当前任务类型。

## 安全使用边界

Hub 文档和对话只保留在工作流层面：

| 应该做                                                                                | 避免做                                               |
| ------------------------------------------------------------------------------------- | ---------------------------------------------------- |
| 使用 `suncode hub init`、`login`、`state`、`intake`、`plan-ready`、`review`、`finish` | 把内部服务细节、凭据或敏感值贴进 prompt              |
| 在团队内部私下共享项目 ID 和 Hub 位置                                                 | 把团队专用 Hub 配置写进公开文档或公开示例            |
| 团队任务动作前先看 `hub state`                                                        | 对普通本地任务运行 Hub 提交或状态同步动作            |
| 私人工作继续使用本地 Suncode task                                                     | 因为项目配置了 Hub，就把所有 task 都当作团队同步任务 |

## 常见情况

| 情况                   | 处理方式                                                                                                    |
| ---------------------- | ----------------------------------------------------------------------------------------------------------- |
| Hub 没有启用           | 继续使用本地 Suncode；如果团队使用 Hub，再运行 `suncode hub init`。                                         |
| 还没有登录             | 运行 `suncode hub login`，然后再运行 `suncode hub state`。                                                  |
| Hub 当前不可连接       | 非团队工作可以继续本地推进；团队工作等服务恢复后再处理。                                                    |
| 有可处理团队工作       | 单个需求运行 `suncode hub intake --auto`；多个需求先选 ID，再运行 `suncode hub intake --requirement <id>`。 |
| 当前任务显示为本地任务 | 除非明确把它绑定到团队工作，否则按本地任务完成。                                                            |
